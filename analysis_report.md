**작성일:** 2026-07-12  
**프로젝트 경로:** `iran_oil_app/`

---

## 1. 프로젝트 개요

### 주제 및 목적

이란-이스라엘 전쟁(2026년 2월~) 이후 글로벌 유가 급등이 한국 1인 가구(자취생)의 월 생활비에 미치는 영향을 정량적으로 분석하고, 세 가지 전쟁 확전 시나리오(국지전·호르무즈 봉쇄·전면전)에 따른 생활비 변화를 예측하는 대시보드를 제공한다.

단순한 물가 정보 제공을 넘어, 사용자가 **자신의 실제 소비량을 입력하면 시나리오별 예측 추가 부담을 계산**할 수 있도록 설계하였다.

### 개발 배경

2022년 러시아-우크라이나 전쟁에서 브렌트유가 배럴당 117달러까지 급등했던 사례에서 확인되듯, 중동 분쟁은 에너지 가격을 통해 식품·교통·공과금 등 생활 전반에 파급 효과를 낳는다. 특히 에너지 수입 의존도가 높은 한국에서는 유가 상승이 소비자물가에 빠르게 전달된다.

본 프로젝트는 통계청 생활물가지수 데이터와 야후 파이낸스 브렌트유 시세를 결합하여, 유가 → 물가 전달 경로를 OLS 회귀분석으로 실증하고 그 결과를 웹 인터페이스로 시각화하였다.

| 실시간 브렌트유 | 야후 파이낸스 `BZ=F` | 최근 5거래일 | 앱 시작 시 1회 수집 |

#### 데이터 파일 3-tier 폴백 전략

배포 환경에 로컬 파일이 없어도 앱이 정상 동작하도록 다음 순서로 대체한다.

```
① 로컬 CSV/Excel 존재 → 파일 로드
② 파일 없음 → yfinance API 다운로드
③ API 실패 → 하드코딩 기준값 사용 (2022-01~2026-06 수작업 추정)
```

### 전처리 과정

1. **Excel 파싱**: 통계청 Excel은 헤더 없는 raw 형식. `header=None`으로 읽은 후 컬럼 위치로 직접 접근  
   - 열 위치 2~54번 → 2022-01 ~ 2026-05 (53개월)  
   - 열 헤더는 float(`2022.01`)으로 저장되어 있어 `_parse_month_col()` 함수로 `"YYYY-MM"` 문자열로 변환  
2. **월별 브렌트유**: CSV의 일별 데이터를 `groupby("month").last()`로 월말 종가 집계  
3. **공통 월 추출**: 브렌트유 시계열과 물가지수 시계열에서 `set` 교집합으로 겹치는 월만 분석에 사용  
4. **결측 처리**: `np.isfinite()` 마스크로 NaN/Inf 제거 후 회귀 수행

### OLS 회귀분석 방법

브렌트유 유가의 월별 변화율이 각 품목 물가지수의 월별 변화율에 미치는 영향을 **단순 OLS 회귀**로 추정한다.

**모형:**

$$\Delta P_{i,t} = \alpha_i + \beta_i \cdot \Delta O_t + \varepsilon_{i,t}$$

- $\Delta O_t$ : 브렌트유 월별 % 변화율 (독립변수)
- $\Delta P_{i,t}$ : 품목 $i$의 물가지수 월별 % 변화율 (종속변수)
- $\beta_i$ : 유가 탄성계수 (추정 대상)
- $\alpha_i$ : 절편 (fit_intercept=True)

**구현 코드 요약:**

```python
o_pct = np.diff(oil_prices) / oil_prices[:-1]   # 브렌트유 월별 변화율
p_pct = np.diff(item_vals)  / item_vals[:-1]     # 품목 물가 월별 변화율

model = LinearRegression(fit_intercept=True)
model.fit(o_pct.reshape(-1,1), p_pct)

beta  = model.coef_[0]     # 탄성계수 원시값
r2    = model.score(...)   # 결정계수
```

샘플 수 n < 6이면 회귀를 건너뛰고 사전 추정치를 사용한다.

### 탄성계수 계산 방법

OLS로 산출된 $\beta_i$ (원시값)를 아래 기준으로 최종 탄성계수로 확정한다.

| 조건 | 적용 탄성계수 |
|---|---|
| R² ≥ 0.05 | 회귀 계수 $\beta_i$를 `[0.01, 1.5]`로 클리핑 후 사용 |
| R² < 0.05 | 사전 도메인 추정치 유지 (전문가 판단값) |

**시나리오 가격 예측 공식:**

$$P_{pred} = P_{now} \times (1 + \Delta_{oil} \times \beta_i)$$

- $\Delta_{oil}$ : 시나리오별 유가 상승률 (국지전 25%, 호르무즈 60%, 전면전 110%)

### R² 해석

R²(결정계수)는 독립변수(유가 변화율)가 종속변수(품목 물가 변화율) 분산의 몇 %를 설명하는지를 나타낸다. 값이 1에 가까울수록 유가와 물가의 동조성이 높음을 의미한다.

본 분석에서는 **R² < 0.05이면 회귀 계수를 신뢰할 수 없다고 판단**하여 사전 추정치로 대체한다. 이는 식품류 물가에 유가 외에도 계절성·작황·환율 등 다수 요인이 작용하기 때문이다.

## 4. 주요 기능 설명

### 탭별 기능

#### 탭 1: 📊 유가 분석

- **KPI 카드 4개**: 브렌트유 현재가(실시간), 이란 전쟁 이후 상승률, 국내 휘발유 평균가, 러-우 전쟁 최고가 대비
- **브렌트유 라인 차트**: 2022-01~현재 월별 추이, 러-우 전쟁 구간(2022-02~06)과 이란 전쟁 구간(2026-02~)을 음영으로 표시
- 데이터 출처: yfinance `BZ=F` 또는 CSV, 실시간 가격은 앱 시작 시 갱신

#### 탭 2: 🛒 물가 변화

- **KPI 카드 5개**: 라면·달걀·우유·식용유·두부의 2022년 대비 현재 누적 상승률
- **품목별 멀티라인 차트**: 12개 품목의 월별 생활물가지수 추이 (Chart.js, 2020=100 기준)
- **연도별 바 차트**: 2021~2025 연간 평균 지수 비교

#### 탭 3: 🔮 시나리오 예측

- **시나리오 버튼 3개** (국지전/호르무즈 봉쇄/전면전) 클릭 시 즉시 업데이트
- **품목 카드 10개**: 현재가(취소선)와 예측가, 상승률(%)을 카드 형태로 표시
- **시나리오 비교 바 차트**: 시나리오별 월 추가부담 시각화
- API: `GET /api/scenario/{local|hormuz|fullwar}`

#### 탭 4: 🧮 생활비 계산기

- **소비량 직접 입력**: 라면(봉), 달걀(30구), 우유(L), 식용유(병), 생수(병), 샴푸(개), 택배비(건), 휘발유(L), 전기요금(원), 가스요금(원) 등 10개 항목
- **시나리오 드롭다운** 선택 후 계산 실행
- **품목별 예산 변화 테이블**: 현재 예산 → 예측 예산 → 추가부담 → 상승률(%) 표시
- 로그인 시 계산 이력 자동 저장 및 기본 예산 설정 저장 가능
- API: `POST /api/calculate`, `POST /api/history`

## 5. 분석 결과

### 품목별 탄성계수 결과

아래는 `GET /api/analysis` 엔드포인트 반환값 기준 (n=52개월, 2022-01~2026-05).

|---|:---:|:---:|:---:|:---:|:---:|
| **휘발유** | 0.70 | **0.1453** | **0.1338** | ✅ | **0.1453** |
| 식용유 | 0.30 | 0.0733 | 0.0358 | ❌ | 0.30 |

**해석:**
- **휘발유만 R² 기준(0.05)을 통과**하여 회귀 계수(0.1453)가 최종 탄성계수로 채택되었다. 이는 유가가 국내 휘발유 소매가에 가장 직접적으로 반영됨을 의미한다.
- 식품류(라면·달걀·우유 등)는 R²가 매우 낮아(0.006~0.035) 유가 단독으로 가격 변화를 설명하지 못한다. 기상·작황·환율·물류비 등 복합 요인이 작용하는 것으로 판단된다.
- 전기요금·가스요금은 정부 요금 규제로 인해 유가 변동이 즉시 전달되지 않아 상관계수가 낮게 나타났다.
- 회귀 계수가 음수인 항목(라면·달걀·전기요금 등)은 단기 역상관(유가 하락기에 물가가 오르거나, 공급망 지연 등)을 반영한다. R²가 낮으므로 통계적 의미는 없다.

## 6. 개발 과정에서 어려웠던 점

### 1) Chart.js 초기 로딩 문제

탭 전환 시 Chart.js의 `new Chart(canvas)` 가 히든(display:none) 상태의 캔버스에 실행되어 차트가 렌더링되지 않는 문제가 발생하였다. `canvas.getContext('2d')`가 히든 상태에서 0×0 크기를 반환하기 때문이다.

**해결:** 탭1 차트는 "지금 분석 시작하기" 버튼으로 앱이 실제로 화면에 나타난 후 `showApp()` 내부에서 지연 로드(lazy loading)하도록 수정하였다.

### 2) 배포 환경에서의 로컬 파일 의존성

통계청 Excel 파일과 브렌트유 CSV 파일이 로컬에만 존재해 배포 서버에서 앱이 시작되지 않는 문제가 발생하였다. 파일 하나하나가 별도의 장애점(SPOF)이 될 수 있었다.

**해결:** 3-tier 폴백 패턴 (`_load_oil_data()`, `_load_prices_data()`, `_load_cpi_data()`)을 설계하여, 파일 없이도 yfinance API 또는 하드코딩 기준값으로 앱이 구동되도록 하였다. 생활물가지수 폴백은 연구 문헌 및 통계청 보도자료를 참고한 선형보간(`_lp()`) 함수로 구현하였다.

### 3) 유가 탄성계수의 통계적 유의성 문제

식품류 물가에 대한 회귀 결과 R²가 대부분 0.01~0.03에 불과하여, 순수 OLS 계수만 사용하면 오히려 예측 정확도가 하락하는 문제가 있었다.

**해결:** R² 임계값(0.05)을 설정하여 통계적으로 유의하지 않은 경우 사전 도메인 추정치(전문가 판단값)를 유지하는 하이브리드 방식을 채택하였다. 이로써 휘발유처럼 유가와 직접 연동되는 품목은 실증 계수를, 나머지는 보수적 추정치를 사용한다.

### 4) 인증 상태 관리와 UI 동기화

SPA(Single Page Application) 구조에서 토큰 유무에 따라 로그인 버튼·사용자명·마이페이지 탭이 동시에 변해야 하는데, 초기 로드 시 `checkAuth()`가 토큰 없을 때 `updateAuthUI()`를 호출하지 않아 로그인 버튼이 표시되지 않는 버그가 있었다.

**해결:** `if (!token) { updateAuthUI(); return; }` 로 조기 반환 전 UI 업데이트를 명시적으로 실행하도록 수정하였다.

### 5) 한글 인코딩 문제 (Windows 환경)

Windows 터미널(CP949 코드페이지)에서 `python -c` 명령으로 한글 이름을 SQLite에 삽입할 때 인코딩이 깨지는 문제가 있었다. Python의 `sys.stdin`/`sys.stdout` 기본 인코딩이 CP949로 설정되어 UTF-8 한글이 손상되었다.

**해결:** 한글이 필요한 DB 작업은 직접 SQL `UPDATE` 문 또는 Python 스크립트 파일로 실행하고, CI/배포 환경에서는 UTF-8 인코딩을 명시하도록 하였다.
